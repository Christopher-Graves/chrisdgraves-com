# Cloudflare Pages Build Configuration Fix

## Problem
API routes (like `/api/finance/accounts`) return 404 on Cloudflare Pages because the build output directory is incorrect.

## Root Cause
- The site was converted from static export to edge runtime with API routes
- Cloudflare Pages build settings still point to the old `/out` directory
- Should be using `.vercel/output/static` (generated by `@cloudflare/next-on-pages`)

## Fix Required in Cloudflare Dashboard

Go to: **Cloudflare Dashboard → Pages → chrisdgraves-com → Settings → Builds & deployments**

Update these settings:

### Build Configuration
- **Build command:** `npm run build` (should already be correct)
- **Build output directory:** `.vercel/output/static` ← **CHANGE THIS** (was: `out`)
- **Root directory:** (leave empty)

### Environment Variables (Verify These Exist)
- `ADMIN_PASSWORD` - Your admin password
- `DASHBOARD_API_KEY` - Your backend API key
- `NEXT_PUBLIC_API_URL` - Your backend URL (e.g., `https://api.chrisdgraves.com`)

### After Updating
1. Save the settings
2. Trigger a new deployment: **Deployments → View build → Retry deployment**
3. Once deployed, test: `https://chrisdgraves.com/api/finance/accounts`

## What Changed in This Commit
1. **package.json** - Updated default `build` script to use `@cloudflare/next-on-pages`
2. **DEPLOYMENT.md** - Updated deployment guide to reflect edge runtime architecture
3. **README.md** - Fixed deployment instructions
